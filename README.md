# 筑森剪贴 / ZSClip

> 一个更懂本地办公场景的 Windows 剪贴板工具  
> 文本、图片、文件、短语、分组、拖出生成文件、贴图、超级邮件合并，一套完成。

## 为什么是它

`ZSClip` 不只是记录复制历史。  
它更偏向“高频办公操作台”:

- 复制完就能快速粘贴，不用来回切窗口
- 文本、图片、文件都能统一管理
- 常用内容可以沉淀为“短语”
- 支持分组，适合按客户、项目、流程归档
- 支持把列表内容直接拖出为文件
- 支持 Excel -> Word 的“超级邮件合并”

## 功能亮点

### 剪贴板记录

- 自动监听系统剪贴板
- 支持文本、图片、文件
- 主窗口区分“全部记录 / 常用短语”
- 单击即可按设置执行选中、复制或复制后粘贴

### 办公增强

- 快速搜索
- 图片贴图悬浮窗
- 导出为文件
- 文本和图片支持拖出生成文件
- 文件项粘贴走原生文件剪贴板，不是路径文本

### 内容管理

- 常用短语
- 置顶
- 分组管理
- 分组筛选
- SQLite 本地持久化

### 系统集成

- 托盘常驻
- 开机自启
- 全局热键
- 轻量 Win32 原生窗口

### 超级邮件合并

- 读取 Excel 文件
- 识别表头和数据行
- 向 Word 插入 `MERGEFIELD`
- 支持邮件合并模式和填表模式

## 适合谁用

- 经常复制微信、QQ、企业资料、模板文本的人
- 需要反复发送固定回复和标准话术的人
- 经常处理图片、文件、表格、文档的人
- 需要 Excel + Word 联动填充的人
- 希望后续继续做 UI 框架抽离和跨系统移植的人

## 快速开始

1. 运行 `zsclip.exe`
2. 在任意软件里复制文本、图片或文件
3. 使用默认热键 `Win + V` 打开主窗口
4. 点击列表项执行粘贴，或右键执行更多命令
5. 如果当前条目是 Excel 文件，可直接打开“超级邮件合并”

## 常用快捷键

- `Win + V`：显示或隐藏主窗口
- `Ctrl + F`：打开搜索
- `Ctrl + C`：复制当前项
- `Ctrl + P`：置顶或取消置顶当前项
- `Esc`：关闭搜索或收起界面

实际行为可以在设置中调整。

## 超级邮件合并

这是 `ZSClip` 里比较有辨识度的一块能力，适合 Word 模板填充和批量文档处理。

典型流程：

1. 选择 Excel 文件
2. 读取工作表、表头行、数据行
3. 在 Word 中插入字段或填入当前行数据
4. 用于邮件合并模板或普通表单填充

说明：

- 依赖本机 Windows、Excel、Word 环境
- 当前实现优先面向本机 Office 自动化场景

## 运行与构建

建议在 `Developer PowerShell for VS 2022` 或等价的 MSVC 开发环境中执行。

开发运行：

```powershell
cargo run
```

发布构建：

```powershell
cargo build --release
```

或直接使用脚本：

```powershell
.\build_release.ps1
```

## 数据目录

当前版本默认使用程序同级目录下的 `data` 文件夹：

- 配置文件：`data/settings.json`
- 数据库：`data/clipboard.db`
- 导出目录：`data/exports/`

这意味着它天然适合便携使用，复制整个目录即可迁移。

## 项目结构

项目正在按“方便后续移植”的方向整理，主入口逐步收口到：

- `src/ui.rs`：共享 UI 模型、主题、布局与绘制逻辑
- `src/win_system_ui.rs`：Win32 窗口、主题、拖拽、控件宿主等平台适配
- `src/win_system_params.rs`：Win32 常量、控件 ID、平台参数
- `src/app.rs`：应用状态、业务流程、命令分发

目标不是单纯换皮，而是把 UI 逻辑与 Win32 平台层逐步分开，方便后续接别的系统或别的 UI 壳。
